# nginx rollout 실습 이미지 태그 (ch3/3.2.4)

## 변경 내용

| 역할 | 이전 | 이후 | 비고 |
|---|---|---|---|
| 초기 배포 | `nginx:1.23.4` | **`nginx:1.30.1`** | stable 1.30 브랜치의 이른 패치 |
| 정상 업데이트 | `nginx:1.24.0` | **`nginx:1.30.3`** | 같은 stable 브랜치 최신 패치 (초기보다 높음) |
| 실패 테스트 | `nginx:1.25.22` | **`nginx:1.30.99`** | 존재하지 않는 패치 (ErrImagePull 유도) |

## 변경 사유

- stable 브랜치(1.30)로 통일 — 프로덕션은 stable을 쓰는 게 일반적이라 입문서 실습에 적합. mainline(1.31)은 개발 브랜치라 제외
- 초기(1.30.1) → 업데이트(1.30.3): 같은 stable 안에서 패치만 올리는, 실무에서 가장 흔한 업데이트 시나리오
- 실패용 1.30.99: 기존 1.25.22(1.25.2 오타 컨셉)와 동일한 목적 — 존재하지 않는 패치 버전으로 이미지 pull 실패(ErrImagePull) → 롤백 실습 유도
