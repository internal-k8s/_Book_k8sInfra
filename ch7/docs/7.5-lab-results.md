### 테스트 2: 한국어 — "쿠버네티스의 파드(Pod)란 무엇인지 2문장으로 설명해주세요."

| 모델 | 품질 | 비고 |

### 관찰 결과

- 영어: 3개 모델 모두 사용 가능한 답변을 생성하지만 품질 편차가 있음
- 한국어: 심각한 품질 저하 — 환각, 지시 불이행, 다국어 혼란 발생
- 각 모델의 강점/약점이 언어와 주제에 따라 다름
- **단일 sLLM만으로는 신뢰할 수 없음**, 특히 비영어 콘텐츠에서 두드러짐

## 7.5.3 MoA (Mixture of Agents) — K=1 vs K=3 비교

### 알고리즘

```
K=1:  [질문] → sLLM → 답변
      (단일 모델, 교차 검증 없음)

K=3:  [질문] ─┬─ sLLM-1 (Gemma3)  → 답변 A ─┐
               ├─ sLLM-2 (Qwen3.5) → 답변 B ─┼─ Aggregator → 최종 답변
               └─ sLLM-3 (Llama3.2) → 답변 C ─┘
               (3개 모델이 독립적으로 답변, Aggregator가 최선의 내용을 종합)
```

- **Step 1**: 동일한 질문을 3개 모델에 전달
- **Step 2**: 3개 응답을 Aggregator 모델에 전달
- **Aggregator 프롬프트**: "3개 답변의 가장 좋은 부분을 종합하고, 잘못된 정보는 제거하세요."

### 결과 1: 영어 — "What is Kubernetes and why is it important?"

| 구성 | 결과 |
|------|------|
| **K=1** (Llama 단독) | 정확하지만 한 모델의 관점과 학습 편향만 반영 |
| **K=3** (3개 모델 + Aggregator) | 3개 답변의 핵심을 종합 — 더 균형 잡히고 포괄적인 답변 |

영어에서는 K=1, K=3 모두 사용 가능한 답변을 생성. K=3이 더 다듬어졌지만 차이는 보통 수준.

### 결과 2: 한국어 — "Docker와 Kubernetes의 차이점을 설명해주세요."

| 구성 | 결과 |
|------|------|
| **K=1** (Llama 단독) | **심각한 환각**: "Docker는 Visual Studio Code에 포함된 플랫폼", "Dockersolve"라는 허구 용어 생성 — 완전히 오답 |
| **K=3** (3개 모델 + Aggregator) | Gemma와 Qwen이 정확한 정보(컨테이너 가상화 vs 클러스터 스케줄링)를 제공하여 Llama의 환각을 교정. 일부 다국어 혼재가 있지만 **사실적으로 훨씬 정확** |

한국어 비교에서 K=1 → K=3 간 **극적인 개선**이 확인됨. Aggregator가 Gemma/Qwen의 정확한 답변과 교차 참조하여 Llama의 환각을 성공적으로 필터링.

### 핵심 시사점

1. **다양한 관점이 개별 모델의 약점을 보완**
   - Gemma: 한국어 이해력 약함, 간결한 영어에 강함
   - Qwen: thinking 과정이 노출되지만 기술적으로 정확한 내용 제공
   - Llama: 영어에 강하지만 한국어에서 심각한 환각
   - 종합 시: 각 모델의 오류가 다른 모델의 정확한 답변으로 필터링됨

2. **MoA 효과는 작업 난이도에 비례**
   - 쉬운 작업 (영어 Q&A): 소폭 개선
   - 어려운 작업 (한국어, 복합 추론): 대폭 개선

3. **Aggregator 품질도 중요**
   - Aggregator 역시 한계가 있는 sLLM
   - 그럼에도 참고할 3개 답변이 있으면, 처음부터 혼자 답변하는 것보다 나은 결과를 생성

## 실행 참고 사항

- 모든 `kubectl` 명령에 `--context=kubernetes-admin@kubernetes` 필수 (다중 클러스터 구성 시 자동 context 전환 방지)
- sLLM 이미지 Pull에 모델당 5~10분 소요 (모델 가중치가 컨테이너 이미지에 내장)
- MoA Pod 완료까지 3~8분 소요 (모델 응답 지연 시간에 따라 변동)
- 테스트 환경: VirtualBox 7.2, Apple Silicon (ARM64), Ubuntu 24.04
