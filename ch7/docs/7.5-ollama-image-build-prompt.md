# 7.5 Ollama 커스텀 이미지 멀티플랫폼 빌드

## 목적
3개 sLLM 모델이 미리 포함된 Ollama 이미지를 빌드한다.
학생이 `kubectl apply` 한 번으로 모델 다운로드 없이 바로 사용할 수 있게 하기 위함.

## 빌드할 이미지 목록

| 이미지 | Ollama 모델명 | 크기 |
|--------|-------------|------|
| `sysnet4admin/ollama-qwen3.5:0.8b` | `qwen3.5:0.8b` | ~1.0GB |
| `sysnet4admin/ollama-gemma3:270m` | `gemma3:270m` | ~291MB |
| `sysnet4admin/ollama-llama3.2:1b` | `llama3.2:1b` | ~668MB |

## 플랫폼
- `linux/amd64` (Intel/AMD 노트북)
- `linux/arm64` (Apple Silicon 노트북)

## Dockerfile (공통 패턴)

```dockerfile
FROM ollama/ollama:latest

# 모델을 이미지에 미리 포함 (bake)
RUN ollama serve & \
    sleep 5 && \
    ollama pull MODEL_NAME && \
    kill %1

# 컨테이너 시작 시 ollama serve 실행 + 자동으로 모델 로드
EXPOSE 11434
CMD ["serve"]
```

## 빌드 & 푸시 명령어

Docker Buildx 멀티플랫폼 빌드를 사용한다.

### 사전 준비
```bash
# Docker Hub 로그인
docker login -u sysnet4admin

# buildx 빌더 생성 (한 번만)
docker buildx create --name multiplatform --use
docker buildx inspect --bootstrap
```

### 빌드 & 푸시
```bash
# 1. Qwen3.5-0.8B
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -f Dockerfile.qwen35 \
  -t sysnet4admin/ollama-qwen3.5:0.8b \
  --push .

# 2. Gemma3-270M
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -f Dockerfile.gemma3 \
  -t sysnet4admin/ollama-gemma3:270m \
  --push .

# 3. Llama3.2-1B
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -f Dockerfile.llama32 \
  -t sysnet4admin/ollama-llama3.2:1b \
  --push .
```

## HF 토큰 관련

- **Ollama pull은 HF 토큰 불필요** (Ollama 자체 레지스트리 사용)
- Gemma3도 Ollama 레지스트리에서 받으므로 Gated 문제 없음
- HF 토큰은 vLLM 참고용 이미지 빌드 시에만 필요 (이번 빌드에서는 불필요)

## 빌드 후 검증

```bash
# amd64 테스트
docker run --rm -d --name test-ollama -p 11434:11434 sysnet4admin/ollama-qwen3.5:0.8b
curl http://localhost:11434/api/chat -d '{
  "model": "qwen3.5:0.8b",
  "messages": [{"role": "user", "content": "Hello"}],
  "stream": false
}'
docker stop test-ollama

# arm64 테스트 (Apple Silicon에서)
# 동일 명령어 — 자동으로 arm64 이미지 pull
```

## 주의사항

1. **빌드 시간**: 각 이미지당 모델 다운로드가 포함되므로 10-30분 소요
2. **이미지 크기**: 모델이 bake되므로 ollama base(~1GB) + 모델 크기
   - qwen3.5:0.8b → 약 2GB
   - gemma3:270m → 약 1.3GB
   - llama3.2:1b → 약 1.7GB
3. **멀티플랫폼**: QEMU 에뮬레이션으로 빌드하므로 cross-platform 빌드 시 느림
   - amd64 머신에서 arm64 빌드, 또는 그 반대 시 2-3배 느림
4. **기존 이미지 참고**: `sysnet4admin/ollama-gemma2:2b` 등 기존 패턴과 동일
